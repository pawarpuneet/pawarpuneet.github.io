How to implement Polynomial Regression and Pipelining?

Let's find out what are the different ways of implementing polynomial regression with one dimension (single feature), multiple dimensions (multiple features) and how can we make all the steps concise with pipelining.

Puneet Pawar Sat 01 November 2025 Sat 01 November 2025
machine_learning sklearn numpy regression polynomial pipelining

Polynomial Regression and Pipelining¶

Let's perform the following:

  1. Polynomial regression with one dimension
  2. Polynomial regression with multiple dimensions
  3. Pipelining

Polynomial Regression with multiple dimensions¶

Example: for two dimensions(two features $X_1$ and $X_2$) the model looks like: $$ Yhat = a + b_1 X_1 +b_2 X_2 +b_3 X_1 X_2+b_4 X_1^2+b_5 X_2^2 $$

NumPy's polyfit function cannot perform this type of regression. We will need to use scikit-learn's preprocessing library to transform features into polynomial features. Let's built a model using these features: 'horsepower', 'curb-weight', 'engine-size', 'highway-mpg'

# define a function to plot predicted vs actual
def plot_pred_vs_actual(y,y_pred):
    fig, (ax1, ax2) = plt.subplots(1,2, figsize=(12,6))
    sns.histplot(y,fill=False, kde=True, label='Actual', ax=ax1)
    sns.histplot(y_pred, fill=False, kde=True, label='Predicted',ax=ax1)
    ax1.legend()
    ax1.set_title('Distribution of price (predicted vs actual)')

    ax2.scatter(y,y_pred, c='g')
    lims = [
        np.min([ax2.get_xlim(), ax2.get_ylim()]),
        np.max([ax2.get_xlim(), ax2.get_ylim()])
    ]
    ax2.plot(lims,lims, c="#C8CAC8")
    ax2.set_xlabel('Actual')
    ax2.set_ylabel('Predicted')
    ax2.set_xticks(np.linspace(5000,45000,9))
    ax2.set_yticks(np.linspace(5000,45000,9))
    ax2.set_title('Price (predicted vs actual)')
    plt.show()

Pipelining¶

There are many steps to getting a prediction. For example, polynomial transform, normalization, and linear regression. We can simplify the process using a pipeline. Pipeline allows you to sequentially apply a list of transformers to preprocess the data and, if desired, conclude the sequence with a final predictor for predictive modeling. Intermediate steps of the pipeline must be transformers, that is, they must implement fit and transform methods. The final estimator only needs to implement fit.

We create a list of tuples. The first element in the tuple contains the name of the estimator, model. The second element contains model constructor. We input the list in the pipeline constructor.

Let's use pipline for steps we performed above and see if we get same results.

In [20]:
%%capture
X = df[['horsepower', 'curb-weight', 'engine-size', 'highway-mpg']]
y = df['price']

# create a list of tuples for all the steps in format (model name, constructor)
# step1: create polynomial features
# step2: normalize the data
# step3: linear regression
input = [('pr1', PolynomialFeatures(degree=2,include_bias=False)), ('scaler1', StandardScaler()), ('lr1',LinearRegression())]
pipe = Pipeline(input)
pipe.fit(X,y)

We got the same results with pipelining as we got by performing every step individually. Pipelining make it much more concise.

Using shorthand for pipeline (without naming the estimators)¶

make_pipeline is a shorthand for the Pipeline constructor; it does not require, and does not permit, naming the estimators. Instead, their names will be set to the lowercase of their types automatically.

Conclusion:¶

We have performed:

  1. 1 dimension polynomial regression (degree 3 and degree 11) using numpy polyfit
  2. For multidimensional data (multiple features) we used sklearn's preprocessing to generate polynomial features and then performed linear regression using those transformed features.
  3. Repeated step 2 but used pipelining to make all transformations concise.
