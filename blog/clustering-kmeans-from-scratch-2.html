# Assigning clusters
def calculate_distance_assign_clusters(df,centroids):
    df['dist_centroid_1'] = df.apply(lambda row: calculate_distance(row,centroids.iloc[0]),axis=1)
    df['dist_centroid_2'] = df.apply(lambda row: calculate_distance(row,centroids.iloc[1]),axis=1)
    df['cluster'] = df.apply(lambda row: 1 if row['dist_centroid_1'] < row['dist_centroid_2'] else 2, axis=1)

    return df

Computing centroid means¶

We have our first cluster split. However, the K-means algorithm consists of multiple iterations until the centroids converge to the mean of their clusters.

We'll perform the next step in this iteration by creating new centroids and assigning clusters for a second time.

As the name of the algorithm suggests, the new centroids are calculated by the mean of each of the K clusters. Once we have the Age and Spending Score coordinates from the new centroids, the process will repeat:

  • Calculate the distance of each customer to the new centroids.
  • Assign new clusters to the customer based on the new distances.
  • Visualize the new clusters.

def create_clusters(df):
    init_centroids = df.sample(n=2)
    df = calculate_distance_assign_clusters(df,init_centroids)
    new_centroids = df.groupby('cluster').mean()[['age','spending_score']].reset_index(drop=True)
    df = calculate_distance_assign_clusters(df,new_centroids)

    return df['cluster']

K-Means Implementation (n iterations)¶

In this version, the code will cluster the data not twice, but as many times as necessary for the centroids to converge towards the mean of their clusters — or until it reaches a maximum number of iterations (N) that we'll set. Another important improvement from the first version is that we'll be able to use any number of clusters (K).

Finally, in the new implementation, we'll still only work with two variables; however, we'll write more flexible code that does not rely on the variables' names, allowing us to use any combinations of features they want to.

# Calculating distance
def calculate_distance(df,centroids_coords):
    for idx,centroid in enumerate(centroids_coords):
        df['dist_centroid_'+str(idx+1)] = np.sqrt((df.iloc[:,0] - centroid[0])**2 + (df.iloc[:,1] - centroid[1])**2)
    return df, [col for col in df.columns.to_list() if 'dist_centroid_' in col]

Using the number of iterations as a parameter is very important in K-means, because depending on the first (and random) initialization of clusters, the algorithm may take a long time to stop.

This number of iterations is the maximum number of iterations because the algorithm also has another condition to stop running and will not necessarily need to run all the iterations.

The other condition is when the centroids don't change from one iteration to another. When centroids don't change, it means that they are already the mean of their cluster and, therefore, won't change again, which is the condition to stop the iterations.

K-Means Cost Function (Intertia/Distortion function)¶

One of the most important metrics for evaluating a K-means algorithm is inertia. Inertia reflects how far the data points assigned to a cluster are from that cluster's centroid. Mathematically, inertia is the sum of the squared Euclidean distances from each of m data points to it's assigned centroid, as given by the following formula:

$$Inertia = \sum_{i=1}^m ||x_{i} - c_{xi}||^2$$

where:

  • $x_{i}$ is a data point
  • $c_{xi}$ is the centroid assigned to $x_{i}$

Distortion, on the other hand, is the average of the squared distances from each data point to its cluster centroid. So, $$Distortion = Inertia/m$$ where:

  • m is the number of data points

Notice that the formula uses the squared distance not only to keep all the numbers positive, but also to make sure that the more distant a point is to the cluster, the more the cluster will be penalized. This means that a cluster with some points very close to the centroid and some points very far from the centroid will present higher inertia than a cluster with points located at a medium distance from the centroid. Also, Inertia and the number of clusters are inversely proportional, which means that the lower the number of clusters, the higher the inertia.The goal is to find the sweet spot in the inertia-clusters trade-off. We need to find the lower number of clusters that produces the lowest inertia and still makes sense to split the data with.

Let's update the kmeans funtion to calculate inertia as well.

# kmeans with inertia calculation
def kmeans(df,k,max_iter,plot=False):
    variables = df.columns.to_list()
    centroids_df, centroids_coords = get_centroids(df,k)
    for i in range(max_iter):
        # Calculating distances
        last_coords = centroids_coords
        df, df_dist_cols = calculate_distance(df,centroids_coords)
        # Assigning cluster
        df['cluster'] = df[df_dist_cols].idxmin(axis=1)
        df['cluster'] = df['cluster'].str.split('_').str[-1]
        # Recalculating Centroids
        centroids_df = round(df.groupby('cluster').mean()[variables],4)
        centroids_df = centroids_df.reset_index(drop=True)
        centroids_coords = centroids_df.values.tolist()
        if last_coords==centroids_coords:
            break
    # identify distance column based on assigned cluster and square the distance
    df['sqrt_dist_centroid'] = df.apply(
    lambda row: row[f'dist_centroid_{row["cluster"]}'] ** 2, axis=1
    )
    # calculate interia
    inertia = df['sqrt_dist_centroid'].sum()

    print(f'Num of iterations: {i+1}')
    if plot:
        sns.scatterplot(df,x=variables[0],y=variables[1],hue='cluster', s=50)
        sns.scatterplot(centroids_df,x=variables[0],y=variables[1],s=100,c='black')
        plt.show()
    return df, inertia

Number of Clusters and Elbow Curve¶

The result of inertia was a big number--over 183 thousand. But over 184 thousand what? As we're not using any particular unit of measurement, we're calculating the distance between units of annual income and units of spending score, we can say that the inertia was over 183 thousand units of distance.

But this number doesn't make much sense by itself. In order to better understand this number, we'll compare it to other inertias on the same data.

The 183 thousand inertia refers to the split of the customers dataset between two clusters. The next step is to calculate the inertia for other numbers of clusters, which will make us capable of comparing and deciding whether or not 183 thousand is a big number and what a reasonable number for inertia should be.

The inertia is decreasing as the number of clusters increase. That makes perfect sense with what we discussed before: if the number of clusters keeps increasing and reaches the number of observations in the dataset, then inertia will be zero.

But when is it enough? At which point should we say that the inertia has decreased to a point where we're comfortable using that number of clusters?

Keeping in mind that the ideal model has few clusters and low inertia, we can see the number of clusters as the cost of lowering inertia. We should stop adding clusters when the inertia reduces at lower and lower rates. In economics, this is called diminishing returns.

We call it the elbow curve because we assume that the first clusters will cause the inertia to decrease very fast, and, at some point, this decrease will become slower in such a way that we'll see a sharp "elbow" on the curve. This sharp "elbow" is where we should set our number of clusters. The bottom line is that the Elbow Curve allows us to visually understand how the number of clusters affects the quality of the model. A single value for inertia does not say anything, but a visual representation of many inertias put together makes it a lot easier to set the number of clusters for our model.

In the plot above, the elbow isn't very sharp, but a broader elbow made of three points: 3,4,5. After k=5, the decrese in inertia is not significant and the line becomes almost horizontal.

This plot shows how much the inertia decreases (in percentage terms) as $ K $ increases. Notice that until k=5, the decrease is much higher, and it's going up and down, which means it's not decreasing at lower and lower rates. From the fifth to the sixth point, the decrease is dramatically lower.

Important thing is, clustering is usually related to business and operations areas in companies, since we're frequently looking to segment customers, stores, products, purchases, and more. So we need input from business teams when making decisions about the ideal number of clusters.

Let's say we're not segmenting customers, but stores in a big chain of supermarkets in our city. The company has built five distribution centers, strategically located around the city, and, although a distribution center can service up to two clusters, they all need to have at least one cluster assigned to them. This means we would need a minimum of five clusters and by looking at the curve, we would definitely choose five.
